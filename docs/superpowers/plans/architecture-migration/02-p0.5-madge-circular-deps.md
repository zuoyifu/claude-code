## Task 3: 分类循环依赖

**Files:**

- [ ] **Step 1: 创建 baseline 文档骨架**

Create `docs/superpowers/refactor-assets/circular-deps-baseline.md`:

```markdown
# 循环依赖基线（P0.5 完成时）

- **生成时间：** 2026-07-11（P0.5 实施时）
- **工具：** madge 8.x
- **扫描范围：** `src/` 下所有 `.ts` / `.tsx` 文件
- **总循环数：** N（待填）

## 分类

### 类型 A：工具系统循环（C1 必须优先处理）

| # | 循环 | 涉及文件 | 解除方案 |
|---|------|---------|---------|
| A1 | Tool.ts → tools.ts → TeamCreateTool → Tool.ts | 3 文件 | 见下方方案 |

### 类型 B：query 循环（C9 实施时处理）

| # | 循环 | 涉及文件 | 解除方案 |
|---|------|---------|---------|

### 类型 C：services 循环（不在本次范围）

| # | 循环 | 涉及文件 | 备注 |
|---|------|---------|------|

### 类型 D：其他

## 解除方案模板

### 方案 1：依赖注入

适用：A 类（tools.ts ↔ TeamCreateTool）

**当前：**
\`\`\`
src/tools.ts:69: const getTeamCreateTool = () => require('...TeamCreateTool.js')  // 懒加载破环
src/TeamCreateTool.ts: import { X } from '../../tools.js'  // 反向引用
\`\`\`

**目标：**
- TeamCreateTool 不再反向 import `tools.ts`
- 改为通过 `ToolUseContext` 注入所需依赖

**步骤：** C1 Task N 中实施。

### 方案 2：抽取共享模块

适用：两个文件互相 import 同一个 helper

**当前：** A ↔ B 都 import `validateTool()`，定义在 A 中
**目标：** 抽 `validateTool` 到 `core/validation.ts`，A 和 B 都依赖 core

### 方案 3：事件机制

适用：UI 组件 ↔ 业务逻辑循环

**当前：** Component ↔ Service 互引
**目标：** Service emit 事件，Component subscribe

## 验证

C1 完成后跑 `bunx madge --circular src/`，预期 A 类循环数 → 0。
```

## Risk

| 风险 | 严重度 | 缓解 |
|------|--------|------|
| madge 解析 `bun:bundle` 等 Bun 特有 import 失败 | 高 | 用 `--ts-config tsconfig.json` + madge 的 `--exclude` 排除 `bun:*`；若仍失败，改用 `madge --webpack-config` 模式 |
| 循环数太多（>50），文档巨长 | 中 | 分类时只详述 A/B 类（阻断项），C/D 类只列计数 |
| 发现的循环超出预期，C1 工程量爆炸 | 高 | 这是 P0.5 的核心价值——**提前发现**。若 A 类循环 >10 个，需重新评估 C1 可行性 |
| graphviz 未装导致 SVG 失败 | 低 | 已在 Step 4 处理 |

  ```js
    required: ['totalCycles', 'classA', 'classB']
  }
  ```
- **可并行点:** 无（P0.5 内部串行）。
